Development Philosophy

Core principles and architectural decisions that make ZEN Starter exceptional

Modular TypeScript Architecture

Class-based component system with proper dependency injection. Each component is self-contained with clear interfaces.

import { Header } from '@/components';
const header = new Header('#header', options);

Responsive Design Principles

Mobile-first approach with fluid typography using rem units. CSS custom properties for consistent spacing and breakpoints.

:root {
--spacing-xs: 0.5rem;
--spacing-lg: 2rem;
}

Performance-First CSS

PostCSS optimization pipeline with media query combining. Critical CSS extraction and unused rule elimination.

Accessibility by Default

ARIA attributes, keyboard navigation, and semantic HTML structure. Screen reader compatibility and focus management.

<button aria-expanded="false">
<span class="sr-only">Toggle menu</span>
</button>

Service Layer Pattern

Reusable service modules for common functionality. API abstraction, storage management, and utility functions.

class StorageService {
static get(key: string): any { ... }
static set(key: string, value: any): void { ... }
}

Responsive Typography with rem()

SCSS rem() function for consistent scaling across breakpoints. Automatic pixel-to-rem conversion with base font size consideration.

// SCSS rem() function
font-size: rem(16px); // 1rem
padding: rem(24px); // 1.5rem
// Automatically scales with root font-size

Fluid Scaling with fluid()

CSS fluid() function for smooth scaling between breakpoints. Eliminates the need for multiple media queries for typography.

// Fluid typography scaling
font-size: fluid(16px, 24px, 320px, 1200px);
// Smoothly scales from 16px to 24px

Component-Based SCSS Structure

Modular SCSS architecture with component-specific files. BEM methodology and consistent naming conventions for maintainability.

// Component-based SCSS
.button {
&--primary { @extend .button; }
&--secondary { @extend .button; }
}

Accessibility & Semantic HTML

Proper heading hierarchy, landmark regions, and ARIA labels. Keyboard navigation support and screen reader optimization.

<main role="main">
<section aria-labelledby="section-title">
<h2 id="section-title">Section</h2>
</section>
</main>

Mobile-First Responsive Design

Progressive enhancement approach starting from mobile devices. Touch-friendly interactions and optimized performance for small screens.

// Mobile-first media queries
.element {
padding: 1rem; // Mobile default
@media (min-width: 768px) {
padding: 2rem; // Tablet+
}
}
